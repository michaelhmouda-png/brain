### UUID Columns Only

1. **id** (PRIMARY KEY)
   - Auto-generated: `gen_random_uuid()`
   - Not included in INSERT statement (PostgreSQL generates automatically)
   
2. **company_id** (REQUIRED FK)
   - No default value
   - Cannot be null
   - Must exist in companies table
   - Included in INSERT
   
3. **department_id** (OPTIONAL FK)
   - Nullable
   - No default value
   - When null, omitted from INSERT via conditional spread `...(deptId ? {...} : {})`
   - When present, must exist in departments table
   
4. **location_id** (OPTIONAL FK)
   - Nullable
   - No default value
   - When null, omitted from INSERT via conditional spread `...(locId ? {...} : {})`
   - When present, must exist in locations table

### Foreign Key Constraints

- `company_id → companies(id)` ON DELETE CASCADE
- `department_id → departments(id)` ON DELETE SET NULL
- `location_id → locations(id)` ON DELETE SET NULL

### Triggers on Employees Table

**Only ONE trigger defined:**
- Name: `employees_update_timestamp`
- Event: BEFORE UPDATE
- Action: Executes `update_timestamp()` function
- Function: Sets `new.updated_at = now()`
- **IMPORTANT:** This trigger fires only on UPDATE, NOT on INSERT

### Functions Called by Triggers

`update_timestamp()` function:
```sql
create or replace function update_timestamp()
  returns trigger
  language plpgsql
as $$
begin
  new.updated_at = now();
  return new;
end;
$$;
```

### RLS Policies on Employees Table

**INSERT Policy:** `employees_insert`
- Condition: `private.can_manage_company(company_id)`
- Called functions: `can_manage_company()` → checks role + company_id match
- Called by: `is_super_admin()`, `is_active_user()`, `current_user_role()`, `current_user_company_id()`

**UPDATE Policy:** `employees_update` 
- Uses/Check: `private.can_manage_company(company_id)`

**DELETE Policy:** `employees_delete`
- Uses: `private.can_manage_company(company_id)`

**SELECT Policy:** `employees_select`
- Allows viewing own company employees or super_admin sees all

### Potential UUID Error Sources — Eliminated

1. ✅ **id column** — Not included in INSERT, auto-generated by PostgreSQL
2. ✅ **company_id** — Confirmed valid UUID in pre-insert logs
3. ✅ **department_id** — Null, so omitted from INSERT via conditional spread
4. ✅ **location_id** — Null, so omitted from INSERT via conditional spread
5. ✅ **No DEFAULT values on UUID columns** — Only id has default (gen_random_uuid())
6. ✅ **No INSERT triggers** — Only UPDATE trigger exists (employees_update_timestamp)
7. ✅ **RLS policy functions** — Don't modify UUID values, only check permissions

### Remaining Mystery

All UUID columns are accounted for. The only INSERT-time UUID validation happens via RLS policies, which call:
- `private.can_manage_company(company_id)`

This function receives company_id parameter and compares it. It doesn't generate or modify UUID values.
